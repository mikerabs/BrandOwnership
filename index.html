Database Structure

Table Definitions

Categories Table

Category_ID: Integer. The unique identifier for each category. Primary Key.

Category_Name: Varchar. The name of the category, such as "Fitness", "Personal Care", or "Food".

Description: Text. A brief description of what the category entails.


Subcategories Table

Subcategory_ID: Integer. The unique identifier for each subcategory. Primary Key.

Category_ID: Integer. Foreign Key that references Category_ID in the Categories table. Indicates the main category to which the subcategory belongs.

Subcategory_Name: Varchar. The name of the subcategory, such as "Vitamins", "Supplements" within the Fitness category.

Description: Text. A brief description of what the subcategory entails.


Brands Table

Brand_ID: Integer. The unique identifier for each brand. Primary Key.

Brand: Varchar. The name of the brand.

Owner: Varchar. The company or entity that owns the brand.

Ownership Type: Varchar. The classification of the owner, such as 'Founder owned', 'Family Owned', or 'Megacorp'.

Notes: Text. Any relevant notes about the brand or its ownership.


BrandSubcategoryJunction Table

scj_id: Integer. The unique identifier for each record in this junction table. Primary Key.

Brand_ID: Integer. Foreign Key that references Brand_ID in the Brands table.

Subcategory_ID: Integer. Foreign Key that references Subcategory_ID in the Subcategories table.


Interconnectivity

The Categories and Subcategories tables are interconnected through a one-to-many relationship. Each category can encompass multiple subcategories, but each subcategory is exclusive to one category, as indicated by the Category_ID in the Subcategories table.

The Brands table is linked to the Categories table via a one-to-many relationship. Each brand is associated with one category, which is shown by the CategoryID in the Brands table.

A many-to-many relationship exists between the Brands and Subcategories tables, facilitated by the BrandSubcategoryJunction table. This setup allows a brand to be linked to multiple subcategories and vice versa. The junction table uses BrandID to reference the Brands table and SubcategoryID to reference the Subcategories table.
